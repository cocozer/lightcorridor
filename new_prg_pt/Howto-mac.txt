A priori OpenGL est préinstallés directement dans MacOS X
Pour gLfw voir sur glfw.org 

Il faut mieux compiler directement GLFW à l'aide d'un cmake

Sinon la documentation recommande ceci : 
> It is recommended that you use pkg-config when building from the command line on macOS. 
> That way you will get any new dependencies added automatically. 
> If you still wish to build manually, you need to add the required frameworks and libraries to your 
> command-line yourself using the -l and -framework switches.

> If you are using the dynamic GLFW library, which is named libglfw.3.dylib, do:
    cc -o myprog myprog.c -lglfw -framework Cocoa -framework OpenGL -framework IOKit

> If you are using the static library, named libglfw3.a, substitute -lglfw3 for -lglfw.

> Note that you do not add the .framework extension to a framework when linking against it from the command-line.

Attention, il faut aussi modifier les include dans les fichiers gldrawing.h et visu.c
> remplacer les #include <GL/...> par #include <OpenGl/...>
